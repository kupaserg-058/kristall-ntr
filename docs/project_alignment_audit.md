# Аудит соответствия MVP материалам проекта

Дата: 19.06.2026

Ветка: `feature/ntr-result1-alignment`

## 1. Что сейчас реализовано в репозитории

MVP `Кристалл.НТР` уже содержит рабочий базовый контур:

- backend на FastAPI;
- PostgreSQL через SQLAlchemy async;
- frontend на React/Vite;
- загрузку документов `.pdf` и `.docx`;
- регистрацию федеральных документов из папки `federal_docs/` при старте backend;
- запуск анализа документа через Gemini 2.5 Flash;
- извлечение направлений НТР с фрагментами-основаниями и уровнем уверенности;
- список документов со статусами обработки;
- общий список направлений с фильтрами;
- ручную экспертную верификацию направлений: подтверждено, требует уточнения, отклонено;
- статистику по документам, направлениям и статусам проверки.

Текущая версия соответствует первой технической версии MVP в части сценария:

`загрузить или выбрать документ -> обработать документ -> получить перечень направлений НТР -> проверить результат экспертом`.

## 2. Соответствие результату 1

По пакету `Результат_1` первая контрольная работа закрывает две линии:

1. формирование пилотного набора стратегических и отраслевых документов;
2. подготовку эталонной экспертной разметки направлений НТР и исходного ручного замера.

Текущий репозиторий частично соответствует этой логике.

| Требование результата 1 | Статус | Комментарий |
|---|---|---|
| Папка федеральных документов для MVP | подтверждено | Федеральный корпус перенесен в `federal_docs/`. |
| Загрузка и обработка PDF/DOCX | подтверждено | Реализовано через `/api/documents/upload` и `/api/documents/{id}/analyze`. |
| Извлечение направлений НТР | частично подтверждено | Извлечение направлений реализовано; в ветке `feature/ntr-result1-alignment` README, промпт Gemini и frontend-список направлений переведены на Указ N 529. |
| Экспертная проверка направлений | подтверждено | Есть ручная смена статуса направления. |
| Эталонная разметка / gold standard | не подтверждено | В приложении пока нет сущности эталонной разметки, сравнения с эталоном и расчета TP/FP/FN/F1. |
| Исходный ручной замер | не подтверждено | В приложении нет журнала ручного времени и протокола baseline. |
| Пилотный набор 5 региональных документов | не подтверждено | Федеральный корпус добавлен, региональный пилотный корпус в репозиторий пока не перенесен. |

## 3. Соответствие паспорту и плану управления

В плане управления результат 1.1 формулируется как первая версия MVP с анализом документов и извлечением направлений НТР до 30.06.2026.

Текущий MVP закрывает:

- базовый пользовательский интерфейс загрузки или выбора документов;
- программный компонент обработки документов;
- формирование перечня направлений НТР;
- ручную экспертную проверку найденных направлений.

Пока не закрывает:

- обработку именно пилотного набора региональных стратегий;
- выгрузку из ГИСП;
- исходный замер ручного подбора организаций;
- подбор предприятий и научных организаций;
- отображение оснований подбора организаций;
- расчет проектных показателей: сокращение времени, F1, релевантность организаций, объяснимость рекомендаций.

Это нормально для текущей стадии, если считать репозиторий первой технической сборкой, а не полным результатом проекта.

## 4. Соответствие первой главе диплома

Первая глава описывает более широкий демонстрационный сценарий:

`выбор/загрузка документа -> извлечение направлений НТР -> выбор направления -> сопоставление с предприятиями и научными организациями -> отображение оснований рекомендации -> экспертная оценка -> выгрузка результата`.

Текущий репозиторий реализует первую половину сценария:

- выбор/загрузка документа;
- извлечение направлений;
- экспертная оценка найденных направлений.

Вторая половина сценария пока отсутствует:

- каталог организаций;
- сопоставление направлений с организациями;
- причины включения организации в рекомендацию;
- выгрузка результата;
- расчет сокращения времени ручного подбора.

Следовательно, текущий MVP хорошо подходит как версия 1, но не должен описываться как полный MVP из первой главы.

## 5. Соответствие научной статье

Статья строит методику вокруг федерально-региональной согласованности и Указа N 529. В ней важны:

- извлечение приоритетов НТР из документов;
- проверка связи с направлениями Указа N 529;
- технологическая конкретизация;
- проекты и меры;
- организации и кооперация;
- KPI и управляемость;
- фиксация разрывов.

Текущий MVP покрывает только первый уровень:

- извлекает направления;
- сохраняет цитату-фрагмент;
- позволяет эксперту подтвердить или отклонить результат.

Пока не покрыты уровни, которые делают методику статьи полной:

- технологии;
- проекты;
- организации;
- KPI;
- типы разрывов;
- оценка по шести критериям.

## 6. Главный методологический риск

В исходной версии README и системной инструкции Gemini было указано, что федеральные приоритетные направления берутся из Указа N 145 от 28.02.2024.

По материалам проекта базовым классификатором должен быть Указ Президента РФ от 18.06.2024 N 529:

- именно он задает 7 рабочих направлений для MVP;
- по нему должна строиться эталонная разметка;
- по нему должен считаться F1;
- остальные документы должны выполнять вспомогательную роль.

В ветке `feature/ntr-result1-alignment` это исправлено: пользовательская документация, системный промпт Gemini и frontend-фильтр направлений используют формулировки Указа N 529.

## 7. Технические риски текущей сборки

| Риск | Статус | Комментарий |
|---|---|---|
| Порт БД в `docker-compose.yml` и дефолтном `DATABASE_URL` | требует проверки | Docker публикует PostgreSQL на `5433`, а дефолтный `DATABASE_URL` смотрит на `5432`. Локальный запуск требует `.env` с правильным портом или изменения дефолта. |
| Нет миграций БД | частично подтверждено | Таблицы создаются через `Base.metadata.create_all`; для MVP допустимо, для дальнейшей разработки лучше добавить миграции. |
| Федеральные `.rtf` не анализируются приложением | подтверждено | Startup и upload поддерживают только `.pdf` и `.docx`; `.rtf` сохранены в корпусе, но не будут автоматически зарегистрированы как документы. |
| Нет тестов | подтверждено | В репозитории не обнаружены backend/frontend тесты. |

## 8. Рекомендуемый порядок следующих PR

1. Добавить структуру результата анализа под методику проекта: метка 529, региональные формулировки, технологии, проекты, организации, KPI, разрывы.
2. Добавить пилотные региональные документы и/или отдельный реестр пилотного корпуса.
3. Добавить сущность эталонной разметки и расчет F1 по направлениям 529.
4. Добавить baseline-журнал ручного замера времени.
5. После этого переходить к каталогу организаций и объяснимому подбору.

## 9. Вывод

Репозиторий уже является рабочей первой технической сборкой MVP для анализа документов и экспертной проверки извлеченных направлений.

Для дальнейшего соответствия результату 1, первой главе и научной статье нужно расширить модель данных от простого списка направлений к проверяемому профилю документа: направления 529, технологии, проекты, организации, KPI, разрывы и доказательные фрагменты.
